Pact 服务能力架构图

从 Agent Harness 和 Pact Client 向上展示 MCP Plugin、统一操作路由、缓存、网关和外部服务目标，兼容性隐含在系统边界和网关能力中。

视角：MCP Plugin To External Services 状态：工作稿 层级：自下而上

MCP Plugin To External Services

Layer 6 外部应用
代码库
GitHub
  • Repo、Issue、Pull Request、Review。
  • 通过统一 Repo Gateway 接入。
Gerrit
  • Review、Patch Set、Submit、Abandon。
  • 保留 Route Decision、Audit、Projection。
外部知识库
Dify
  • Dataset、Document、Segment。
  • 面向 Agent Workflow 的知识检索。
RAGFlow
  • RAG Engine、Dataset、Chunk。
  • 文档解析、Embedding、召回与 Rerank。
云盘 / 文件源
iCloud
  • 文件镜像。
OneDrive
  • 增量同步。
Google Drive
  • 共享文件。
Dropbox
  • 对象拉取。
↑
Layer 5 网关层
Repo Gateway
  • 统一 Repo Operation。
  • Provider = GitHub / Gerrit。
Knowledge Gateway
  • External Knowledge Adapter。
  • 返回 Evidence Root 和 Content Refs。
Cloud Drive Gateway
  • 文件列表、增量同步、下载。
  • 统一进入 LSM Ingest / CAS。
Block Sync Gateway
  • Missing Blocks Discovery。
  • 按 Root / CID 传输缺失块。
Secret And Connection Gateway
  • Secret Ref、Token、连接配置。
  • 不把外部凭据暴露给智能体。
↑
Layer 4 缓存与算法底座层
LSM Ingest Pipeline
  • Upload WAL、MemTable、Immutable Segment。
  • 只管理 session / chunk metadata。
CAS Block Store
  • Content Addressed Blocks。
  • 文件主体、Evidence、Artifact 不可变。
Merkle DAG Manifests
  • File、Context、Evidence、Artifact Root。
  • 支持去重、缺块发现和校验。
Prefix / Range Index
  • Context Prefix Reuse。
  • Metadata Range Diff。
State Commit / Projection
  • beforeRoot、afterRoot、contentRefs。
  • Checkpoint Tree 是可重建视图。
↑
Layer 3 统一操作路由层
Operation Router
  • 统一接收 pact.call / HTTP / RPC / CLI。
  • 解析 operationId、subject、workspace、params。
Policy Router
  • 统一处理 Grant、Scope、Risk、Confirmation。
  • 写入 Audit、Denied Reason 和 Execution Envelope。
Capability Router
  • 分派到 Knowledge、Sharedspace、Codespace、Skill Hub。
  • 结果进入 Ledger、StateCommit 和 Projection。
↑
Layer 2 MCP 插件能力层
Discovery
  • 能力发现、工具说明、Doctor。
  • 返回可用命令和连接状态。
Knowledge
  • Search、Evidence、Asset、Export。
  • 受 AgentLibrary 权限裁决。
Sharedspace
  • Context、Files、Artifacts、Proposals。
  • 共享状态受 StateCommit 管理。
Codespace
  • 代码空间、代码库状态、Diff、Commit。
  • GitHub / Gerrit Provider 接入。
Skill Hub
  • Skills、Tools、Toolsets。
  • 统一走 Grant、Risk、Policy 和 Audit。
↑
Layer 1 客户端类型
Agent Harness
  • OpenClaw、Codex、Claude Code、Cursor Agent。
  • 第三方 Agent Framework。
  • 机器人或自动化系统。
  • MCP Compatible Agent Harness。
Pact Client
  • pact CLI。
  • Shell / CI Script。
  • pact-mcp-connector。
