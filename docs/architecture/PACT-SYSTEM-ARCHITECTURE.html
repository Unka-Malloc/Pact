Pact 系统架构图

从管理层到基建层展示 Pact 内部系统结构，重点表达接口封装、应用能力、公共基础设施和侧边启动装配边界之间的关系。

历史基线：2026-05-19 状态：工作稿 范围：Pact 内部结构

管理层

Management
  • 提供知识库、工作空间、调试和系统管理界面。
  • 只调用服务层接口，不直接访问应用层运行时或基建层存储。

注入管理层入口

Composition Root

装配 Console shell、管理页面和只读系统视图。

  • 只注入服务层入口。
  • 不暴露应用 runtime 或基建存储。
服务层

接口封装层

Interface Wrapper
  • 统一 HTTP / RPC / CLI 三种交互。
  • 负责认证、权限、参数归一化和返回格式。
  • 对上承接管理层，对下只调用装配后的能力。

注入接口适配器

Composition Root

注册 HTTP / RPC / CLI / MCP route、handler、adapter。

  • 注入认证和权限裁决端口。
  • 注入应用层 operation/provider。
