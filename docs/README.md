# SplitAll Docs

This directory contains project-level architecture, operation, and design
documents.

- `Architecture.md`: client/server architecture and module boundaries.
- `PROTOCOLS.md`: protocol ownership, upstream/downstream placement, and
  protocol/implementation separation.
- `SERVER.md`: server startup, CLI, protocol, packaging, and operations.
- `ENTITY-CONFIG-LAYOUT.md`: human-maintainable entity config directories,
  lightweight skill bundles, and verification.
- `USAGE.md`: console and client workflows.
- `CLOUD-BOUNDARY.md`: cloud/local responsibility boundary.
- `KNOWLEDGE-DISTILLATION.md` and `KNOWLEDGE-METABOLISM.md`: knowledge
  workflow notes.
- `DESIGN.md`: console design notes.
- `GIT-COLLAB.md`: local collaboration hygiene.

Generated files do not belong in `docs/`. Promote only stable documentation
that should be reviewed and kept.
