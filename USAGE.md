# SplitAll 使用说明

这份说明面向实际使用者，重点解释如何启动程序、如何配置云端智能体、如何使用奇安信浏览器模式，以及如何导出结果。

## 1. 程序用途

SplitAll 用来把以下输入材料：

- 直接粘贴的文本
- 本地拖拽或选择的文件
- 图文混合材料

处理为以下输出：

- 带时间戳的最小知识文档
- 带时间戳的模拟问答对
- 可导出的 `JSON`、`Markdown`、`Word (.docx)`

如果输入中包含图片，程序会把图片一起送入云端模型分析；导出 Word 时会附带原始图片内容。

## 2. 启动方式

程序支持两种模式。

无论是哪种模式，实际处理都是“提交后台任务 -> 页面轮询状态 -> 完成后加载结果”，不是前台同步卡住等待。

### 2.1 桌面模式

这是默认模式。程序会使用 Electron 自带的 Chromium 内核运行页面，不依赖目标机器已有浏览器。

开发环境启动：

```bash
npm install
npm run dev
```

打包后的程序：

- Windows：直接双击启动
- 麒麟 V10：直接运行安装后的应用，或执行 `AppImage`

### 2.2 奇安信浏览器模式

这个模式下，程序会：

1. 启动本地 HTTP 服务
2. 自动拉起奇安信浏览器
3. 在奇安信浏览器中打开本机页面

开发环境启动：

```bash
npm run browser
```

打包后启动：

- Windows：

```bash
SplitAll.exe --browser
```

- 麒麟 V10：

```bash
./SplitAll.AppImage --browser
```

如果浏览器没有被自动拉起，请先在设置中填写“奇安信浏览器路径”。

## 3. 首次使用前的配置

程序左侧有一组配置项，首次使用至少要填下面三项。

### 3.1 API Base URL

填你的云端智能体接口地址。当前实现默认按 OpenAI 兼容的 `/chat/completions` 协议调用。

示例：

```text
https://api.openai.com/v1
```

如果你们有自己的兼容网关，也可以填你们自己的地址。

### 3.2 API Key

填你的访问密钥。

### 3.3 模型

填你要调用的模型名。

示例：

```text
gpt-4.1-mini
```

如果要稳定处理图文混合材料，请使用支持视觉输入的模型。

### 3.4 Tika JAR 路径

程序已经内置 `tika-app-3.2.3.jar`，默认不需要填写这个字段。

只有在你想替换成自己的 Tika 版本时，才需要手工填写完整路径。

示例：

```text
/opt/splitall/vendor/tika/tika-app-3.2.3.jar
```

### 3.5 Java 路径

程序已经内置 `Temurin JRE 17.0.18`，默认不需要填写这个字段。

只有在你想替换成自己的 Java 运行时，或者临时指向另一套 JRE 时，才需要手工填写完整路径。

示例：

```text
/opt/splitall/vendor/jre/linux-x64/bin/java
```

### 3.6 奇安信浏览器路径

这个字段主要给浏览器模式使用。

留空时，程序会尝试常见路径和常见命令名，例如：

- Windows：`qaxbrowser.exe`
- 麒麟 / Linux：`qaxbrowser-safe-stable`、`qaxbrowser-safe`

如果自动探测失败，请直接填写奇安信浏览器可执行文件的完整路径。

示例：

- Windows：

```text
C:\Program Files\Qaxbrowser\Application\qaxbrowser.exe
```

- 麒麟 V10：

```text
/usr/bin/qaxbrowser-safe-stable
```

### 3.7 奇安信浏览器参数

这个字段用于附加启动参数。

麒麟 V10 上如果浏览器要求禁用 sandbox，可填：

```text
--no-sandbox
```

如果不需要额外参数，可以留空。

### 3.8 系统提示词

程序已经内置了一份默认提示词，用于要求模型输出：

- 最小知识单元
- 模拟问答对
- 强制附带时间戳
- 严格返回 JSON

一般情况下可以直接使用默认值。只有在你们对切分粒度或字段结构有特殊要求时，才需要改。

配置完成后，点击“保存连接配置”。

## 4. 输入材料的方法

程序支持两类输入，可以混用。

### 4.1 粘贴文本

把原始材料直接粘贴到右侧“粘贴文本”区域即可。

适合：

- 会议纪要
- 文档片段
- 制度说明
- FAQ 草稿

### 4.2 文件拖拽或选择

可以把文件拖进页面，也可以点“选择文件”。
如果你有整批资料，也可以直接点“选择文件夹”，或者把整个文件夹拖进页面。

当前支持：

- 文本：`txt` `md` `markdown`
- 结构化文本：`csv` `json` `yaml` `yml`
- 文档：`docx` `pdf` `doc` `ppt` `pptx` `xls` `xlsx` `rtf` `msg` `eml` `epub` 以及常见 ODF 格式
  这些格式默认直接走程序内置的 Tika 解析链
- 图片：`png` `jpg` `jpeg` `webp` `gif`

注意：

- 桌面模式下，文件按本地路径读取
- 奇安信浏览器模式下，文件通过浏览器上传到程序自己的本地服务处理
- 选择文件夹或拖入文件夹时，程序会递归扫描子目录，只抓取支持的类型，自动忽略不支持的文件

## 5. 生成结果

完成输入后，点击“开始切分”。

提交后，按钮会进入“后台处理中...”状态。你可以把窗口放在后台，或在浏览器模式下刷新页面；如果任务还在运行，页面会自动继续查询该任务状态。

程序会把内容送到云端智能体，返回两类结果：

### 5.1 知识文档

每条知识文档包含：

- 标题
- 来源
- 正文
- 标签
- 时间戳

### 5.2 模拟问答

每条问答包含：

- 问题
- 答案
- 来源
- 关联知识单元标题
- 时间戳

所有时间戳都使用 ISO-8601 格式。

## 6. 结果导出

右上角支持三种导出。

### 6.1 导出 JSON

适合系统对接、二次处理、导入知识库。

### 6.2 导出 Markdown

适合人工阅读、文档归档、版本管理。

### 6.3 导出 Word

适合发给业务人员、项目存档、正式文档流转。

如果输入中包含图片，Word 导出会带上原图附件内容。

## 8. 常见问题

### 8.1 为什么桌面模式还会带 Chromium？

因为程序要求“不能假设目标机器上有现成基础环境”，所以桌面模式必须自带前端运行时。Electron 会把 Chromium 和 Node 一起打包进去。

### 8.2 为什么奇安信浏览器模式还需要这个程序先启动？

因为浏览器模式不是把前端页面单独扔到公网服务器，而是由本程序在本地启动服务，再通过奇安信浏览器访问这个本地页面。文件读取、导出、配置保存、调用云端模型，仍然由本程序负责。

### 8.3 自动拉不起奇安信浏览器怎么办？

优先按这个顺序排查：

1. 在设置中填写奇安信浏览器完整路径
2. 如果是麒麟 V10，尝试增加 `--no-sandbox`
3. 在目标机器上确认奇安信浏览器确实能从命令行直接启动

### 8.4 导出 Word 失败怎么办？

先确认：

- 结果已经成功生成
- 输入图片不是损坏文件
- 目标导出位置有写权限

### 8.5 哪些内容不会被处理？

当前不会稳定支持以下内容：

- 受密码保护的 PDF
- 极大体积的二进制文件
- 不可识别编码的特殊文本文件
- 非常规私有格式文档

## 9. 推荐使用流程

推荐按下面顺序使用：

1. 启动程序
2. 填写云端智能体配置
3. 如果要走浏览器模式，填好奇安信浏览器路径
4. 粘贴文本或拖入文件
5. 点击“开始切分”
6. 在“知识文档 / 模拟问答”标签下检查结果
7. 选择需要的格式导出
